KNOX code challenge

The KNOX team imports data from many sources. The data doesn't always follow a consistent format. In this challenge you will write a program that reads in data in various formats and outputs the data as a sorted csv.

Input

You will receive three input files from three ad exchanges

  • Ads-R-Us: AdCampaignName, Impressions, Clicks, TotalSpend, date(dd-mm-yyy)
  • BannerMart: date(mm/dd/yy) | AdCampaignName | Clicks | Impressions | TotalSpend
  • TrippleClick: Impressions Clicks TotalSpend AdCampaignName date(yyyymmdd)

Output

Output each row as follows:

ExchangeName, Date, AdCampaignName, Clicks, Impressions, Spend

Order the rows according to the following rules. Rule one takes precedence over rule two etc.

  1. If the number of clicks is greater than 10% of impression list the row first
  2. Date ascending
  3. Exchange Name alphabetical
  4. Ad Campaign Name reverse alphabetical

Requirements

  • Write your solution in Ruby, Python or Node.
  • Use only the standard libraries and test framework of your choice.
  • Include a readme that specifies the programming language and version you used and how to run your program.
  • Include tests.
  • We will be looking for object oriented design, readability, maintainability and effective use of of the language.
